\chapter{Problem Analysis}
\label{chap:problem}
\section{Assumptions}
\begin{itemize}
    \item Disaster struck zone: Partially down communications, BLE first, mesh
    \item Politically oppresive regimes: various trust levels, edge rendezvous points, p2p
    \item Online monitoring: encryption through Noise handshake for key exchange
    \item More?
\end{itemize}

\section{Design Goals}
\begin{itemize}
    \item Prefer local over remote communication
    \item When available, use the Internet-scale relation to relay messages over long distances
    \item Reduce attack interface for surveillance and monitoring as much as possible: expose the minimum amount of required information to facilitate communication
    \item Store-carry-forward: cache packets for unavailable recipients until a possible delivery attempt can be made
    \item Do not ignore existing mobile infrastructure to facilitate Internet-scale communication: edge rendezvous points and hole-punching
    \item Because the application may be used in various circumstances, let the user decide on various security-critical settings (i.e. use cloud RV point, friends, introduction facilitation)
\end{itemize}
