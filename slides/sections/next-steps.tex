\begin{frame}{Since last weekly}
    \begin{itemize}
        \item Migrated BLE communincation layer from two separate central/peripheral plugins to one \code{grassroots\_bluetooth\_layer}: unified event model, shared discovery UUID, post-connect ANNOUNCE identity, and BLE role toggle.
        \item Reworked UDX into a dual-stack Internet transport: IPv4/IPv6 sockets, multi-candidate addresses, live network/public-address refresh, and best candidate-pair selection.
        \item Aligned rendezvous/NAT traversal with GLP Networking API doc: two-sided punch, RV fan-out (A and B try each others' RV servers when the connection breaks), friends-of-friends mediation (friends can facilitate rendezvous for mutual friends).
        \item Persist friend data in encrypted local storage so that connections can be initiated post restart.
        \item Added Noise XX handshake for secure channel encryption: messaging and signaling with Noise XX across clients and rendezvous anchors.
        \item Hardened delivery over UDX/BLE: ACKs after fragment reassembly, increase max UDX payload size for photos, and stale-session/socket recovery (app loses connections when backgrounded)
    \end{itemize}
\end{frame}

\begin{frame}{Internet-scale UDX: possible extensions}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.49\textwidth}
\small
\textbf{\textcolor{TUMBlue}{Transport and discovery}}
\begin{itemize}
  \setlength\itemsep{0.25em}
  \item \textbf{BLE discovery optimization:} energy-aware scan/advertise duty cycling without losing too many encounters.
  \item \textbf{Dual-transport selection:} choose BLE or UDX per message based on reachability, latency, trust, cost, and payload size.
  \item \textbf{Cost-aware routing:} quantify bytes, battery, retries, relay load, and latency per delivered message.
\end{itemize}

\vspace{0.4em}
\textbf{\textcolor{TUMBlue}{Measurement}}
\begin{itemize}
  \setlength\itemsep{0.25em}
  \item \textbf{Well-connected friends:} characterize which peers can realistically help with signaling and rendezvous.
  \item \textbf{Grassroots NAT study:} measure NAT behavior, address churn, IPv4/IPv6 availability, and hole-punch success.
\end{itemize}
\end{column}

\begin{column}{0.49\textwidth}
\small
\textbf{\textcolor{TUMOrange}{Trust and identity}}
\begin{itemize}
  \setlength\itemsep{0.25em}
  \item \textbf{Out-of-band identity exchange:} peer links should carry intent/attestation, not only a public key.
  \item \textbf{Friend-based relay:} safest relay mode; reduce attack surface with sealed forwarding and minimal visible metadata.
  \item \textbf{Geographic trust zones:} conferences, workplaces, or classrooms where local strangers may be acceptable relays.
  \item \textbf{Stranger relay:} highest-risk mode, but useful for emergency, protest, or disaster-connectivity scenarios.
  \item \textbf{Relay by trust level:} adapt forwarding policy to friend, trusted-zone, or stranger context.
\end{itemize}
\end{column}
\end{columns}

\vspace{0.4em}
\begin{center}
\small
\textbf{Research question:} once the system is stable, which relay and transport choices improve delivery enough to justify their cost and abuse surface?
\end{center}
\end{frame}
